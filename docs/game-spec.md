# Codex 开发任务：小小王国网页联机版 MVP

你现在是一名资深 TypeScript / Web 游戏开发工程师。请帮助我开发一款网页版多人联机卡牌游戏。

我没有软件开发经验，所以请你在实现代码时保持项目结构清晰、代码注释适量、命令说明明确。（每一份代码前面要说明这份代码的主要功能）

本任务目标是先做出一个可以本地运行和小范围测试的网页 MVP，不需要正式上线，不需要微信小程序适配，不需要账号系统，不需要付费系统。

---

## 一、项目目标

制作一个支持 2-4 人联机游玩的网页卡牌游戏。

第一版 MVP 要实现：

1. 创建房间
2. 加入房间
3. 2-4 人开始游戏
4. 服务端洗牌、发牌
5. 玩家轮流进行回合
6. 每回合先出 1 张牌，再摸 1 张牌
7. 支持放牌到自己的牌列
8. 支持弃牌到公共弃牌堆
9. 支持从牌堆摸牌
10. 支持从弃牌堆顶部摸牌
11. 牌堆摸空后游戏结束
12. 自动计分
13. 显示排名
14. 支持刷新页面后重连
15. 支持基本操作日志

---

## 四、重要开发原则

### 1. 服务端权威

所有关键游戏状态必须由服务端维护和判定。

客户端不能决定：

* 洗牌
* 抽牌结果
* 手牌内容
* 出牌是否合法
* 计分
* 胜负

客户端只负责展示状态和提交操作。

---

### 2. 规则引擎独立

游戏规则逻辑必须独立于 UI。

例如这些函数应该放在 `src/game/` 中：

```ts
createDeck(playerCount)
shuffleDeck(deck)
createInitialGameState(players)
canPlaceCard(column, card)
playCardToColumn(gameState, playerId, cardId, color)
discardCard(gameState, playerId, cardId, color)
drawFromDeck(gameState, playerId)
drawFromDiscard(gameState, playerId, color)
calculatePlayerScore(player)
calculateFinalRanking(gameState)
```

React 组件不要直接实现游戏规则。

---

### 3. 保持可扩展性

虽然 MVP 不实现 buff 和特殊卡牌，但数据结构要预留扩展字段。

例如：

```ts
type StatusEffect = {
  id: string;
  name: string;
  description: string;
  ownerType: "player" | "column" | "room";
  ownerId: string;
  duration?: {
    type: "turns" | "rounds" | "permanent";
    remaining?: number;
  };
};
```

玩家、牌列、房间都可以预留 `statusEffects` 字段，但第一版不需要实现复杂 buff 逻辑。

## 五、游戏规则

## 1. 游戏人数

支持 2-4 人。

房间最多 4 人。

游戏开始后不允许中途加入。

---

## 2. 卡牌颜色

游戏有 5 种颜色：

```ts
type CardColor = "red" | "blue" | "yellow" | "green" | "white";
```

当前先使用抽象颜色名。

未来 UI 可以再映射成正式世界观包装。

---

## 3. 卡牌类型

游戏有 3 种卡牌：

```ts
type CardType = "number" | "multiplier" | "wild";
```

### 数字牌

数字牌有颜色和数字。

数字范围是 2-10。

```ts
type NumberCard = {
  id: string;
  type: "number";
  color: CardColor;
  value: 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9 | 10;
};
```

### 加倍卡

加倍卡有颜色，没有数字。

```ts
type MultiplierCard = {
  id: string;
  type: "multiplier";
  color: CardColor;
};
```

### 万能牌

万能牌没有固定颜色，也没有固定数字。

```ts
type WildCard = {
  id: string;
  type: "wild";
};
```

万能牌放入某个颜色列后，只在该列中生效。

---

## 六、牌堆构成

### 2 人牌堆

5 种颜色。

每种颜色包含：

* 数字牌 2-10 各 1 张
* 加倍卡 3 张

总数：

```txt
5 × (9 + 3) = 60
```

所以 2 人牌堆为 60 张。

---

### 3 人牌堆

在 2 人牌堆基础上增加：

* 万能牌 2 张
* 每种颜色额外增加数字 2、3、4、5 各 1 张

总数：

```txt
60 + 2 + 5 × 4 = 82
```

所以 3 人牌堆为 82 张。

---

### 4 人牌堆

在 3 人牌堆基础上增加：

* 万能牌 1 张
* 每种颜色额外增加数字 6、7、8、9 各 1 张
* 每种颜色额外增加加倍卡 1 张

总数：

```txt
82 + 1 + 5 × 4 + 5 = 108
```

所以 4 人牌堆为 108 张。

注意：之前规则中写过 113 张，那是错误的。代码中请以 108 张为准。

---

## 七、游戏区域

每名玩家拥有：

1. 手牌
2. 自己的 5 个颜色牌列

公共区域包括：

1. 牌堆
2. 5 个颜色弃牌堆

弃牌堆是栈结构，后放入的牌在顶部。

---

## 八、游戏开始流程

1. 玩家创建房间。
2. 其他玩家输入房间号加入。
3. 房间人数达到 2-4 人后，房主可以开始游戏。
4. 服务端根据人数生成牌堆。
5. 服务端洗牌。
6. 每名玩家摸 8 张初始手牌。
7. 随机选择一名玩家作为先手。
8. 游戏进入第一个玩家的出牌阶段。

---

## 九、回合流程

每个玩家回合有 2 个阶段：

1. 出牌阶段
2. 摸牌阶段

当前玩家必须先出 1 张牌，然后摸 1 张牌。

完成摸牌后，回合结束，轮到下一名玩家。

---

## 十、出牌阶段规则

当前玩家必须从手牌中选择 1 张牌，并执行以下操作之一：

### A. 放入自己的卡牌放置区

把牌放入自己的某个颜色列。

必须满足放牌规则。

### B. 放入公共弃牌区

把牌放入对应颜色的弃牌堆顶部。

注意：

* 数字牌可以弃置到对应颜色弃牌堆。
* 加倍卡可以弃置到对应颜色弃牌堆。
* 万能牌不能弃置。
* 万能牌只能放入自己的某个颜色列。

---

## 十一、摸牌阶段规则

当前玩家必须执行以下操作之一：

### A. 从牌堆顶部摸 1 张牌

如果摸到的是牌堆最后一张牌，则该玩家完成当前回合后，游戏结束并进入计分。

### B. 从任意非空颜色弃牌堆顶部拿 1 张牌

玩家只能拿弃牌堆顶部牌。

玩家不能拿回自己本回合刚刚弃置的那张牌。

建议实现方式：

* 在 GameState 中记录本回合刚刚弃置的卡牌 ID，例如 `lastDiscardedThisTurn`
* 如果当前玩家尝试从弃牌堆摸这张牌，则拒绝操作

---

## 十二、放牌规则

每名玩家有 5 个颜色列。

每种颜色最多一个列。

### 1. 新建列

当某颜色尚未存在牌列时，可以放入该颜色的数字牌或加倍卡来建立列。

万能牌也可以放入某个颜色列。

例如：

```txt
红5 -> [5]
```

---

### 2. 数字牌扩展列

向已有列放入数字牌时：

* 新牌颜色必须与该列颜色一致
* 新牌数字必须严格大于当前列最后一个有效数字
* 允许跳号

例如：

```txt
[2][5] + 8 合法
[2][5] + 5 不合法
[2][5] + 4 不合法
```

---

### 3. 加倍卡规则

加倍卡没有数字。

加倍卡只能放在该列出现任何数字之前。

合法：

```txt
[加倍卡][加倍卡][数字牌][数字牌]
```

非法：

```txt
[数字牌][加倍卡]
```

如果某列已经出现数字牌或作为数字使用的万能牌，则不能再放加倍卡。

---

### 4. 万能牌规则

万能牌有两种用途。

### 情况 A：放在数字牌之前

如果万能牌放在某列出现数字之前，则该万能牌视为加倍卡。

例如：

```txt
[万能牌][2][4]
[加倍卡][万能牌][2][4]
```

此时万能牌计入倍率。

---

### 情况 B：放在数字牌之后

如果万能牌放在某列已有数字之后，则该万能牌视为当前最后有效数字 + 1。

例如：

```txt
[2][4][万能牌]
```

这里万能牌视为 5。

后续只能继续放：

```txt
6、7、8、9、10
```

再如：

```txt
[2][5][万能牌]
```

这里万能牌视为 6。

后续只能继续放：

```txt
7、8、9、10
```

---

### 5. 连续万能牌

允许连续万能牌作为数字牌使用。

例如：

```txt
[2][4][万能牌][万能牌]
```

第一张万能牌视为 5。

第二张万能牌视为 6。

后续只能继续放：

```txt
7、8、9、10
```

---

### 6. 10 封顶规则

如果某一列最后有效数字已经达到 10，则该列关闭。

关闭后的列不能再放任何牌，包括：

* 数字牌
* 加倍卡
* 万能牌

---

## 十三、游戏结束条件

当任意玩家从牌堆摸到最后一张牌时：

1. 该玩家完成当前回合。
2. 游戏立即结束。
3. 进入计分阶段。

如果玩家从弃牌堆摸牌，则不会触发牌堆耗尽结束。

---

## 十四、计分规则

每种颜色单独计分。

总分 = 5 个颜色列得分之和。

---

### 1. 未建立的颜色列

如果某个颜色列没有任何牌，则得 0 分。

不扣 20 分。

---

### 2. 已建立的颜色列

计分步骤如下：

### 步骤 1：计算数字和

数字和包括：

* 数字牌的数字
* 作为数字使用的万能牌的实际数字

不包括：

* 加倍卡
* 作为加倍卡使用的万能牌

### 步骤 2：计算基础分

```txt
基础分 = 数字和 - 20
```

### 步骤 3：计算倍率

列首连续加倍卡数量决定倍率。

作为加倍卡使用的万能牌也计入加倍卡数量。

```txt
倍率 = 加倍卡数量 + 1
```

例如：

```txt
0 张加倍卡 -> ×1
1 张加倍卡 -> ×2
2 张加倍卡 -> ×3
3 张加倍卡 -> ×4
4 张加倍卡 -> ×5
```

### 步骤 4：计算列得分

```txt
列得分 = 基础分 × 倍率
```

---

## 十五、胜负判定

游戏结束后，计算所有玩家总分。

总分最高者获胜。

如果多个玩家并列最高分，则并列获胜。

---

## 十六、联机和产品规则

### 1. 房间人数

* 支持 2-4 人。
* 满 4 人后禁止新玩家加入。
* 游戏开始后不允许中途加入。

### 2. 玩家账号

MVP 暂不做账号系统。

玩家只需要输入昵称。

### 3. 刷新重连

需要支持刷新页面后重连。

建议使用：

* roomId
* playerId
* localStorage

如果玩家刷新页面，本地仍有 roomId 和 playerId，则自动尝试重连。

### 4. 掉线处理

第一版支持刷新重连。

如果当前玩家掉线，其他玩家等待。

但每个玩家每回合应该有操作时间限制。

如果当前玩家超时，则自动跳过。

自动跳过的具体策略第一版可以先简单实现：

* 如果处于出牌阶段超时：自动弃掉一张合法可弃置的非万能牌；如果没有可弃置牌，则自动放置一张合法牌；如果都不行，记录错误并跳过。
* 如果处于摸牌阶段超时：优先从牌堆摸牌；如果牌堆为空，则从任意非空弃牌堆摸牌。

如果你认为这个自动跳过策略需要调整，请在实现前说明。

### 5. 聊天功能

MVP 暂不做聊天。

### 6. 观战功能

MVP 暂不做观战。

### 7. 房主退出

游戏开始前：

* 房主退出后，房主身份转移给下一位玩家。

游戏开始后：

* 房主身份不再重要。
* 所有状态由服务端维护。

### 8. 玩家主动退出

游戏开始前可以退出房间。

游戏开始后退出视为掉线，不立即从游戏中移除。
