# 已确认规则决策 v0.1

## 1. 牌堆数量

2 人牌堆：60 张。
3 人牌堆：82 张。
4 人牌堆：108 张。

之前文档中提到的 4 人 113 张是错误的，以 108 张为准。

## 2. 摸回自己弃牌

玩家不能在摸牌阶段摸回自己本回合刚刚弃置的牌。

## 3. 万能牌弃置

万能牌不能弃置，只能放入自己的某个颜色列。

## 4. 万能牌连续作为数字

允许连续万能牌作为数字牌使用。

例如：

```txt
[2][4][万能牌][万能牌]
```

第一张万能牌视为 5。
第二张万能牌视为 6。

## 5. 万能牌作为数字后的后续放牌

万能牌作为数字时，视为当前最后有效数字 + 1。

例如：

```txt
[2][4][万能牌]
```

万能牌视为 5，后续只能放 6、7、8、9、10。

```txt
[2][5][万能牌]
```

万能牌视为 6，后续只能放 7、8、9、10。

## 6. 万能牌颜色

万能牌放入某一颜色列后，只在该列中生效。

后续往该列继续放数字牌时，数字牌颜色必须与该列颜色一致。

## 7. 先手规则

游戏开始时随机玩家先手。

## 8. 平分规则

总分最高者获胜。
如果多个玩家并列最高分，则并列获胜。

排名编号采用竞赛排名：

```txt
1, 1, 3
```

## 9. 只有加倍卡的列如何计分

只有加倍卡、没有数字牌的列属于已建立列。

数字和为 0。

因此得分为：

```txt
(0 - 20) × 倍率
```

这会产生高负分，用于惩罚放了很多加倍卡但没有放足够数字卡的玩家。

## 10. 万能牌放入空列

万能牌放入空列时，立即视为加倍卡。

该作用永久固定，之后不会因为后续放入数字牌而改变。

## 11. 万能牌作用是否永久固定

万能牌的作用按放置时的位置永久固定。

如果放置时位于数字牌之前，则永久视为加倍卡。
如果放置时位于数字牌之后，则永久视为数字牌。

## 12. 无合法出牌时怎么办

正常情况下很少出现无合法出牌。

为了鲁棒性，增加规则：

如果当前玩家在出牌阶段没有任何合法出牌，则跳过出牌阶段，并强制进入摸牌阶段。

## 13. 超时自动操作

每名玩家每次操作有：

```txt
20 秒基础时间 + 50 秒公共额外时间池
```

公共额外时间池属于该玩家本局游戏内的额外时间。

例如某次操作用了 21 秒，则消耗 1 秒额外时间。
下一次操作时间变为：

```txt
20 秒基础时间 + 49 秒额外时间
```

如果玩家某次操作用完了当次可用时间，则判定超时。

重连后不重置计时。

超时自动操作规则：

* 出牌阶段超时：随机弃掉一张非万能牌到对应颜色弃牌堆。
* 如果没有可弃置的非万能牌，则尝试执行合法放置。
* 如果没有任何合法出牌，则跳过出牌阶段并进入摸牌阶段。
* 摸牌阶段超时：优先从牌堆摸牌。
* 如果牌堆为空，该行为视为错误状态。
* 如果牌堆不可摸，则从任意非空弃牌堆摸牌。

## 14. 摸牌阶段牌堆为空

正常流程中，摸到最后一张牌后游戏立即结束，因此不应出现下一位玩家面对空牌堆的情况。

如果异常情况下在摸牌阶段发现牌堆为空，定义为错误行为。

## 15. 游戏结束时是否推进玩家

不推进。

摸到最后一张牌的动作完成后，游戏状态直接变为 `finished`。

当前玩家保留为触发游戏结束的人。

## 16. 日志保存

MVP 阶段完整保存结构化日志。

服务端需要根据玩家视角生成可见日志，避免泄露隐藏信息。

例如：

* 从牌堆摸牌时，其他玩家只能看到“某玩家从牌堆摸了 1 张牌”。
* 不能暴露该玩家具体摸到什么牌。
